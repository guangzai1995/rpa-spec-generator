# RPA 需求规格说明书自动生成系统 — 架构与运行原理

## 1. 系统总览

本系统接收用户填写的 RPA 需求信息 + 业务操作录屏视频，通过 **ASR 语音识别 → 多模态视觉分析 → LLM 结构化拆解 → Word 文档生成** 的自动化流水线，生成一份完整的 RPA 需求规格说明书。

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户浏览器                               │
│   React 18 + TypeScript + Vite                                  │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│   │ 需求表单  │→│ 视频上传  │→│ AI解析中  │→│ 结果预览  │       │
```

## 2. 双模型协作架构

系统采用 **两个独立的 LLM 各司其职** 的设计：

| 职责 | 模型 | 调用方式 | 说明 |
|------|------|---------|------|
| **视觉分析** | Qwen3_5-35B-A3B-FP8 | OpenAI 兼容 API（含 `image_url`） | 分析每帧截图中的 UI 元素、操作意图 |
| **结构化拆解** | minimax-m2.7 | OpenAI 兼容 API（纯文本） | 将表单+ASR+视觉描述融合，生成 JSON 结构 |

```
┌───────────────┐     ┌───────────────┐
│ 关键帧截图     │     │ 表单 + ASR    │
│ (JPEG base64) │     │ + 视觉描述    │
└──────┬────────┘     └──────┬────────┘
       │                     │
       ▼                     ▼
┌──────────────┐     ┌──────────────┐
│  Qwen VL     │     │  minimax     │
│  视觉分析     │     │  结构化拆解   │
│              │     │              │
│ 输出：        │────→│ 输入：        │
│ page_title   │     │ form_info    │
│ page_type    │     │ asr_text     │
│ UI elements  │     │ timeline     │
│ cursor_action│     │ (含视觉描述)  │
│ description  │     │              │
└──────────────┘     │ 输出：        │
                     │ business_    │
                     │   overview   │
                     │ main_process │
                     │ rules        │
                     │ io_spec      │
                     │ exceptions   │
                     └──────────────┘
```

配置方式（`.env`）：

```env
# 文本模型
LLM_API_KEY=xxx
LLM_BASE_URL=https://aicp.teamshub.com/openai/api/v1/openai/v1
LLM_MODEL=minimax-m2.7

# 视觉模型（独立配置，不配则复用 LLM_*）
VISION_API_KEY=yyy
VISION_BASE_URL=https://aicp.teamshub.com/openai/v1
VISION_MODEL=Qwen3_5-35B-A3B-FP8
```

## 3. 核心流水线（Pipeline）详解

### 3.1 流水线全景

```
run_pipeline(requirement_id)
│
├── 0. 加载数据
│   ├── 读取 Requirement + Asset(video) from DB
│   ├── 解析 form_info（用户填写的表单 JSON）
│   ├── 加载 LLM 配置（env → DB fallback）
│   └── 加载 Vision 配置（VISION_* env → LLM_* fallback）
│
├── 1. 预处理 ─────────────── status = preprocessing
│   └── ffprobe → {duration, width, height, fps}
│
├── 2. ASR 转录 ────────────── status = transcribing
│   ├── ffmpeg 提取音频 → 16kHz mono WAV
│   ├── faster-whisper 转录（CUDA→CPU 自动降级）
│   ├── 注入业务术语到 initial_prompt（提升专业词识别率）
│   └── 输出 TranscriptResult{full_text, segments[{start,end,text}]}
│
├── 3. 关键帧提取 ──────────── status = analyzing
│   ├── 计算帧间隔 = max(3s, duration/30)
│   ├── ffmpeg -ss 按时间点截取 JPEG
│   └── 输出 List[FrameInfo{timestamp, path}]（最多 30 帧）
│
├── 3.5 多模态视觉分析
│   ├── 对每帧调用 Qwen VL（最多 15 帧）
│   │   └── 返回 {page_title, page_type, visible_elements, cursor_action, description}
│   ├── 对帧序列调用 Qwen VL 归纳操作步骤
│   │   └── 返回 [{step_no, action, target, context, related_frames}]
│   └── 降级：视觉分析失败不阻塞流水线
│
├── 3.6 保存 TimelineStep
│   ├── ASR 段与关键帧对齐（最近帧 < 8s）
│   ├── 融合视觉分析结果（action/target/context）
│   └── 写入 timeline_step 表
│
├── 4. LLM 结构化拆解 ──────── status = extracting
│   ├── 格式化 timeline 文本（ASR + 视觉描述融合）
│   ├── 调用 minimax（PROMPT_FULL_EXTRACTION）
│   ├── 最多 3 次重试，指数退避
│   ├── JSON 解析（兼容 <think> 块）
│   └── 输出 ExtractionResult{business_overview, main_process, rules, ...}
│
├── 5. Word 文档生成 ──────── status = generating
│   ├── 7 章结构化文档（含截图嵌入）
│   ├── 保存 static/docs/{req_id}_spec.docx
│   └── 写入 spec_doc 表
│
└── 6. 完成 ─────────────── status = success
```

### 3.2 状态机

```
                 ┌─────────────── FAILED ◄──────── 任何阶段异常
                 │
DRAFT ──► SUBMITTED ──► PREPROCESSING ──► TRANSCRIBING ──► ANALYZING
                                                               │
               SUCCESS ◄── GENERATING ◄── EXTRACTING ◄────────┘
```

状态存储在两处（双写，提高轮询性能）：
1. **SQLite `requirement.status` 字段**
2. **文件系统 `note_results/{req_id}.status.json`**（优先读取）

## 7. Whisper ASR 运行机制

```
┌─────────────────────────────────────────────┐
│            transcribe_video()               │
│                                             │
│  1. 检查缓存                                │
│     note_results/{req_id}_transcript.json   │
│     ↓ 命中则直接返回                         │
│                                             │
│  2. 提取音频                                │
│     ffmpeg -i video.mp4 -ar 16000           │
│            -ac 1 -vn audio.wav              │
│                                             │
│  3. 设备检测                                │
│     resolve_whisper_device("cuda")          │
│     ├── ctranslate2.get_cuda_device_count() │
│     ├── GPU 可用 → device=cuda, fp16        │
│     └── GPU 不可用 → device=cpu, int8       │
│                                             │
│  4. 加载模型                                │
│     WhisperModel(                           │
│       "models/pengzhendong/faster-whisper   │
│        -large-v3",                          │
│       device=resolved,                      │
│       compute_type=auto                     │
│     )                                       │
│                                             │
│  5. 转录                                    │
│     model.transcribe(                       │
│       audio.wav,                            │
│       language="zh",                        │
│       initial_prompt="业务术语: XX、YY",     │
│       beam_size=5,                          │
│       vad_filter=True   ← 跳过静音段        │
│     )                                       │
│                                             │
│  6. 降级处理                                │
│     ├── ImportError → 跳过ASR               │
│     ├── 转录异常 → 跳过ASR                   │
│     └── 无有效结果 → 按时长创建占位段          │
│                                             │
│  7. 写入缓存 JSON                           │
└─────────────────────────────────────────────┘
```

## 8. 视觉分析运行机制

### 8.1 单帧分析

```
analyze_frame(image_path)
│
├── PIL 打开图片 → RGB → 缩放(≤1024px) → JPEG(q=85) → base64
│
├── 构造 messages:
│   [{role: "user", content: [
│       {type: "text", text: VISION_PROMPT},
│       {type: "image_url", image_url: {url: "data:image/jpeg;base64,..."}}
│   ]}]
│
├── 调用 Qwen VL API (temperature=0.2, max_tokens=1024)
│
├── 解析响应 JSON：
│   ├── 移除 <think>...</think> 块
│   ├── 尝试 json.loads
│   ├── 尝试提取 ```json ``` 代码块
│   └── 尝试 从 { 到 } 切割
│
└── 返回 {page_title, page_type, visible_elements[], cursor_action, description}
```

### 8.2 序列分析

```
analyze_sequence(frame_analyses[], asr_text)
│
├── 格式化帧分析文本：
│   "[00:00] 截图0: 登录页 - 用户名密码表单"
│   "[00:03] 截图1: 数据列表 - 筛选条件区域"
│   ...
│
├── 填充 SEQUENCE_PROMPT（帧描述 + ASR 文本）
│
├── 调用 Qwen VL API（纯文本模式）
│
└── 返回 steps[{step_no, action, target, context, related_frames}]
```

## 9. LLM 结构化拆解

### 9.1 输入融合

Timeline 文本格式（供 LLM 理解操作上下文）：

```
[00:00] [截图] | 页面:运营数据门户-登录 | 画面:用户名密码输入框 | 操作:输入 | 目标:用户名字段
语音: 首先登录运营数据系统，输入用户名和密码

[00:05] [截图] | 页面:数据查询 | 画面:筛选条件区域 | 操作:选择 | 目标:账期下拉框
语音: 选择T减一的账期进行查询
```

### 9.2 Prompt 架构

```
SYSTEM_PROMPT
├── 角色：RPA 需求分析师
├── 任务：分析 ASR + 关键帧 → 生成结构化需求
├── 输出：中文，严格 JSON
└── 用于所有 LLM 调用的 system message

PROMPT_FULL_EXTRACTION（一次性全量拆解）
├── 输入槽位：{form_info}, {asr_text}, {timeline_steps}
└── 输出：business_overview + main_process + rules + io_spec + system_env + exceptions

备用分步 Prompt（当前未启用，用于 token 不足时分段调用）
├── S1: 概述抽取    → {auto_goal, scope}
├── S2: 流程归并    → {main_process[]}
├── S3: 规则识别    → {rules[]}
├── S4: 输入输出    → {input[], output[]}
├── S5: 系统与权限  → {system_env[]}
└── S6: 异常处理    → {exceptions[]}
```

### 9.3 重试与降级

```
extract()
├── 尝试 1 ──► 成功 → 返回
├── 等待 2s
├── 尝试 2 ──► 成功 → 返回
├── 等待 4s
├── 尝试 3 ──► 成功 → 返回
└── 全部失败 → 返回空 ExtractionResult（文档仍可生成，内容为空）
```

```
封面

├── 需求来源（部门、提出人、日期）
├── 需求背景（scope）
└── 自动化目标（auto_goal）

第2章 业务需求分析
├── 需求类型（网页自动化/桌面自动化/...）
├── 业务场景
│   ├── 执行频率
│   ├── 目标系统 URL
│   └── 是否需要登录
├── 输入来源
└── 输出去向

第3章 系统需求分析
└── 系统表格（名称/认证方式/浏览器）

第4章 功能需求描述             ← 主体内容
├── 流程 A: 系统登录
│   ├── 步骤表格（序号/操作/目标/参数）
│   └── [嵌入截图]
├── 流程 B: 数据筛选
│   ├── 步骤表格
│   └── [嵌入截图]
└── ...

第5章 业务规则与约束
└── 编号列表

第6章 异常处理
└── 异常表格（异常代码/处理方式）

第7章 非功能需求
├── 运行环境：Windows 10+, Chrome 120+
├── 网络：内网 ≥10Mbps
├── 性能：单次 ≤10 分钟
└── 日志：操作日志 + 异常日志
```
